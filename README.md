# 🧲 AEgIS: Trap Chaos of the AD Hall

**A browser-based outreach game about electromagnets and antimatter confinement in the AEgIS experiment at CERN.**

You are an operator in the Antiproton Decelerator (AD) hall. Antiprotons are held in a Penning–Malmberg trap by the magnetic field of a set of superconducting and corrector coils. Your job: tune the coil currents so the axial magnetic field matches the required profile and keep the trapped particles stable before the clock runs out. Detune the field and the trap descends into *chaos* — the particles scatter and confinement is lost.

The magnetic field maps used in the game are generated by the AEgIS **magnetostatic Reduced Order Model (ROM)**, so the physics you play with is the same surrogate model used in the research pipeline — just wrapped in an arcade-style terminal.

---

## ▶️ Play

The game is a single self-contained web page. Because it loads its field data from `magnet_data.json` via `fetch`, it must be served over HTTP — opening `index.html` directly from the file system (`file://`) will fail with a CORS error.

### Option 1 — Run locally

```bash
# clone the repository
git clone https://github.com/Bharat-Singh-Rawat/AEgIS_Electromagnets_Outreach.git
cd AEgIS_Electromagnets_Outreach

# start any static web server, e.g. Python's built-in one
python3 -m http.server 8000
```

Then open **http://localhost:8000** in your browser.

### Option 2 — Host it online

Because it is a static site, it can be published without a backend on:

- **GitHub Pages** — Settings → Pages → deploy from the `main` branch.
- **CERN PaaS** (recommended for CERN-hosted outreach, accessible from outside CERN without a login): https://paas.docs.cern.ch/

## 🎮 How to play

1. **Register** as an operator (name/alias, age, research interest).
2. **Choose a difficulty and mode** on the briefing screen.
3. In the control panel, **adjust the slider for each coil** to change its current.
4. Watch the live plots: the dashed cyan line is the **target** magnetic-field / gradient profile along the trap axis *Z*; the solid line is your **live** field. Bring them together.
5. The **stability / match percentage** rises as your field approaches the target. Keep it high until the timer ends to lock in your score.
6. If the field is badly detuned, the trap enters a **critical state** — the interface flashes red and shakes as confinement collapses.
7. Your final score is saved to the local **Hall of Fame** (Top 5 operators), and you can **export your impact data** as a `.txt` file.

### Difficulty levels

| Level    | Time  | Notes             |
| -------- | ----- | ----------------- |
| Novice   | 120 s | Stable cryostats  |
| Operator | 80 s  | Standard          |
| Expert   | 45 s  | Sensitive coils   |

### Modes

- **Standard Stabilization** — match the nominal AEgIS field profile.
- **Ghost Field Challenge** — a randomized target field each round.

## 🔬 The science behind it

AEgIS confines antiprotons and positrons in a Penning–Malmberg trap, where a strong axial magnetic field provides radial confinement. That field is shaped by two main superconducting solenoids (nominally 5 T and 1 T) together with roughly two dozen corrector/shielding coils that fine-tune the profile and suppress stray fields.

Computing the full field from the real coil geometry is expensive, so AEgIS uses a **Reduced Order Model**: a fast surrogate, built by superposing precomputed unit-current field maps for each coil, that reproduces the field in milliseconds. This game uses exactly that idea — each slider scales a coil's contribution, and the resulting field is summed live — turning the ROM into an interactive teaching tool about how coil currents shape a confinement field.

## 📁 Repository structure

```
AEgIS_Electromagnets_Outreach/
├── index.html        # the entire game — UI, logic, and styling
├── magnet_data.json  # ROM field data: per-coil field maps and the target profile
└── track1.mp3        # background soundtrack
```

`magnet_data.json` contains the axial coordinate grid, the target field profile, the 2-D field grids, and the individual coil field maps (`coils1D`, `coils2D`) that the game superposes as you move the sliders.

## 🛠️ Built with

- **HTML / CSS / vanilla JavaScript** — no build step, no framework.
- **[Plotly.js](https://plotly.com/javascript/)** — real-time field and gradient plots (loaded from CDN).
- **HTML5 Canvas** — particle visualization of the trapped beam.
- **Browser local storage** — persists the Hall of Fame leaderboard between sessions.

> Note: the leaderboard is stored in your browser only; it is per-device and can be cleared from the start screen.
